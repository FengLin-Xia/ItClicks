
《这也能代》PRD V1.1

关系驱动型 AI 同人写作 IDE（Demo）


---

1. 产品定位

1.1 一句话定位

一个以“关系原型”为起点、通过关系操作进行写作协作的 AI 同人写作 IDE Demo。

1.2 核心差异

传统 Chatbot：

从剧情开始

写故事

长文本生成

用户被动阅读


本产品：

从关系结构开始

操作关系

小步生成

用户主导写作



---

2. 目标用户

2.1 核心用户

同人爱好者

代餐读者

想看但不想写长 prompt 的用户

对现有 chatbot 写同人不满意的用户


2.2 用户痛点

1. Chatbot 写文平淡


2. 情绪解释过多


3. 张力不够集中


4. 缺少“记忆点”


5. 写长 prompt 太累




---

3. 产品目标（V1.1）

3.1 用户侧目标

快速触发“这两个人我熟”的感觉

允许用户参与写作

通过关系操作制造爽点


3.2 产品侧目标

构建“关系驱动生成框架”

将生成从剧情层降到关系层

引入关系状态可视化

控制张力与极性分布



---

4. 产品结构


---

Stage 0：关系原型生成页

4.1 页面元素

中央：2–3句关系原型文本

按钮：

换一换

代一代



4.2 关系原型规则

必须满足：

1. 双方立场清晰


2. 关系状态明确


3. 不绑定设定


4. 不解释原因


5. 不出现具体背景


6. 高压缩表达



目标：

> 生成可投射的关系结构，而不是故事片段。

6. 关系状态可视化（V1.1 新增核心）

每次生成不仅输出文本，还输出关系结构数据。


---

6.1 维度一：极性（Polarity）

表示关系张力强度。

范围：

-2 冷

-1 克制

0 中性

+1 偏爱

+2 极端


UI：

横条滑块

当前值高亮



---

6.2 维度二：主动权（Initiative）

表示谁更主导当前互动。

枚举：

A 主导

平衡

B 主导


UI：

左右偏移图标



---

6.3 维度三：关系标签（Tags）

自动生成 1–2 个关系标签：

示例：

偏爱

试探

不对等

口是心非

强撑

明知故犯

早就知道

默契

控制


UI：

Tag Chip 形式展示



---

7. 画布操作区

只保留三个核心操作：


---

7.1 压一层（deepen）

作用：

强化当前关系极性

提高张力

不引入新剧情


输出：

1–2句

不自动插入



---

7.2 换视角（perspective）

作用：

切换表达立场

不改变关系结构


输出：

1–2句



---

7.3 暴露一点（reveal）

作用：

透露隐藏态度

不解释背景


输出：

1–2句



---

8. 生成逻辑架构


---

8.1 后端接口设计

POST /api/generate

Request:

{
  "mode": "seed" | "relation_op",
  "op": "deepen" | "perspective" | "reveal",
  "context": {
    "tail": "最近200-400字文本"
  },
  "state": {
    "polarity": 1,
    "initiative": "A",
    "tags": ["偏爱"]
  }
}


---

8.2 返回格式（V1.1 结构化输出）

{
  "text": "...",
  "relation_state": {
    "polarity": 2,
    "initiative": "B",
    "tags": ["偏爱", "强撑"]
  },
  "meta": {
    "confidence": 0.82
  }
}

9. Prompt 结构设计


---

9.1 Seed Prompt

角色：关系原型生成器

要求：

生成2–3句关系状态

双方立场清晰

不解释

不绑定设定

高压缩



---

9.2 Relation Operation Prompt

角色：关系操作器

根据 op 类型：

deepen：

提高极性

加强互动细节

不写新剧情


perspective：

从另一方立场表达

保持关系不变


reveal：

透露隐藏态度

不解释原因


必须输出：

文本

polarity

initiative

tags
