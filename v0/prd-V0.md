# PRD｜这也能代 v0

## 1. 产品概述

**产品名**：这也能代 v0  
**产品类型**：Web 页面（单页应用）  
**目标**：  
在“无用户数据、极简前端”的条件下，验证一个问题：

> **只通过一段“初段氛围文本”，是否能让用户产生代入感，并愿意继续输入极简 prompt。**

---

## 2. 核心假设（v0 验证点）

1. 用户在 **不提供任何 prompt** 的情况下，看到初段文本，能够：
   - 停留阅读
   - 感觉“有点代”
2. 用户在阅读后，愿意输入**极简补充**（而非完整设定）来继续生成
3. LLM 通过 prompt 工程（而非训练）即可稳定产出“可代入”的初段文本

---

## 3. 用户范围（v0）

- 不做用户区分
- 不做登录
- 不做留存分析
- 不做创作工具定位

**默认用户画像**：
- 能读中文
- 对情绪 / 同人 / 代入有直觉
- 打开即看，不被教育

---

## 4. 功能范围（v0 Only）

### ✅ 包含

1. **打开页面 → 自动生成一段初段文本**
2. **一个 prompt 输入框**
3. **点击生成 → 基于 prompt 生成下一段**

### ❌ 不包含

- 用户系统 / 登录
- 历史记录
- 多段连续管理
- 收藏 / 分享
- 多风格切换
- 数据统计面板

---

## 5. 用户流程（v0）

### Step 1｜进入页面
- 页面加载完成后
- 自动请求一次 LLM
- 返回 **一段初段氛围文本**

用户此时不需要做任何操作。

---

### Step 2｜阅读
- 页面只展示：
  - 产品名
  - 一段文本
  - 一个输入框
  - 一个「继续生成」按钮

目标：让用户先“读”，而不是“想怎么写”。

---

### Step 3｜输入（可选）
- 用户可输入任意简短 prompt，例如：
  - 角色名
  - 一个情绪词
  - 一句否定（“不要和解”）
- 不引导用户写完整设定

---

### Step 4｜生成下一段
- 基于：
  - 上一段文本
  - 用户 prompt（如果有）
- 生成一段 **新的短文本**
- 覆盖或追加显示（v0 可直接覆盖）

---

## 6. 文本生成设计（核心）

### 6.1 初段文本目标

初段文本必须满足：

- 字数：**50–120 字**
- 不出现：
  - 明确剧情事件
  - 角色心理结论
  - 完整故事闭合
- 必须：
  - 有关系张力
  - 有时间 / 距离 / 未发生感
  - 结尾保持悬停

**初段功能定义**：  
> 提供一个“可被代入的情绪与关系占位壳”。

---

### 6.2 Prompt 输入的角色

- Prompt = **轻微方向校准**
- 不是故事说明书
- Prompt 可为空

示例（用户可能输入的）：
- “A 和 B”
- “更痛一点”
- “不要说出口”
- “像告别”

---

## 7. 技术方案（v0）

### 7.1 架构

- 前端：极简 Web 页面
- 后端：单接口（或直接前端调用 API）
- 模型：DeepSeek API
- 无数据库

---

### 7.2 LLM 调用策略（v0 简化版）

#### Writer Prompt（核心）

- 固定系统提示（写作约束）
- 用户 prompt（可空）
- 生成短文本

> v0 **不做 critic / retry**  
> 只验证“是否代得动”

---

### 7.3 Prompt 设计原则（写入系统提示）

- 你是“引线式短文本生成器”
- 输出为中文短文本
- 不解释、不总结、不定设定
- 不写完整事件
- 停在“差一点发生”的位置

---

## 8. 前端设计（极简）

### 页面元素

1. 标题：`这也能代 v0`
2. 文本展示区（居中，偏阅读）
3. 输入框（placeholder 示例：  
   > “你可以只写一个名字或一个感觉”）
4. 按钮：`继续生成`

### 风格建议（非必须）
- 留白
- 行距大
- 无多余 UI 元素
- 不强调“AI”

---

## 9. 成功标准（v0）

### 定性标准（最重要）

- 使用者是否读完初段
- 是否会下意识停顿
- 是否愿意输入一句 prompt

### v0 不追求的指标

- 长时间使用
- 连续生成多段
- 文本“好不好看”

---

## 10. v0 风险与接受项

- 输出偶尔不代 → 接受
- 文本质量不稳定 → 接受
- 用户只看一段就走 → 接受

v0 唯一不能接受的失败是：

> **“第一段完全无法代入，像普通 AI 写作。”**

---

## 11. v0 之后的潜在方向（不做）

- Critic / 重采样
- 多段连续状态
- 氛围切换
- 收藏 / 分享
- 代入强度调节

---

## 12. 一句话总结（给自己看的）

> **v0 的任务不是证明 AI 很会写，  
而是验证：  
“一段克制的文本，能不能点亮用户自己的故事。”**
