《ItClicks 高情绪短文本质量标准 v0.2》
—— LLM 判断版 / 人工判断版拆分稿
0. 拆分原则

当前标准中，部分维度更适合由 LLM 承担初筛与风险识别，部分维度则明显依赖人工语感、风格判断和产品视角。
因此，本版本将原标准拆分为两套：

LLM 判断版：用于自动识别明显 badcase、生成痕迹和结构性问题，适合做批量初筛、自动打标、critic 第一层过滤。
人工判断版：用于判断文本是否真正成立、是否有风格、是否值得进入核心样本池，适合做精选、沉淀和正样本保留。

这一拆分仍然服务于原标准的核心目标，即判断文本是否自然、能代入、有张力、可复用，而不是抽象地评价“文学性高不高”。

一、《ItClicks 高情绪短文本自动评估标准 v0.1（LLM 判断版）》
1. 目标

用于批量识别高情绪短文本中的明显问题，帮助完成 badcase 拦截、样本预筛、自动打标与生成后初步质量过滤。
本版本不负责判断文本是否“高级”或“值得保留”，只负责判断其是否存在明显结构问题或生成痕迹。

2. 适用范围

适用于 ItClicks 中用于刷流展示、定制生成、种子文本筛选的高情绪短文本片段。
适用对象主要是关系瞬间、情绪片段、悬停式表达这类短文本。
不适用于长篇叙事、剧情完整性评价、世界观搭建或文学风格总评。

3. 核心判断维度
3.1 语言通顺度

判断文本是否存在病句、歧义、语义拧巴、表达不自然、人不会这样说的话。
这一维度主要用于拦截明显不通顺或“人话感”不足的文本。

建议标签：

pass
risk
fail
3.2 解释 / 总结过多

判断文本是否把关系、动机、因果和情绪说得太透，导致失去短文本应有的悬停感和余味。
如果文本更像在“讲完一个故事”，而不是留下一个瞬间，则应判为风险项。

建议标签：

none
mild
severe
3.3 设定绑定过重

判断文本是否过度依赖职业、武器、特殊世界观、时代背景等外部设定来支撑气质。
若去掉设定后张力明显塌陷，则说明关系本体不足，属于结构风险。

建议标签：

low
medium
high
3.4 逻辑关联性

判断前后动作、话语、情绪之间是否存在清晰可感知的联系。
若文本试图制造深意，但上下句之间的联系较弱、跳步严重或强行成立，则应标为逻辑风险。

建议标签：

coherent
weakly_coherent
incoherent
3.5 信息支撑是否过空

判断文本是否只有情绪词、狠话、形容词，而缺少关系动作、物理细节或具体支点。
若读者只能感到“情绪很重”，但无法感知张力如何成立，则属于信息过空。

建议标签：

supported
thin
empty
3.6 AI 痕迹 / 标签直译感

判断文本是否存在明显生成痕迹，例如：

把“失衡感、低位感、克制感”等标签直接翻译成句子
高频模板拼贴
明显的“模型在完成任务”感

建议标签：

none
suspicious
obvious
3.7 前史过载 / 戏剧信息过满

判断文本是否为了制造厚度而一次性塞入过多过去关系、创伤信息或戏剧冲突。
若信息密度过高，导致轻推感、留白感和可迁移性下降，则应标为风险项。

建议标签：

light
overloaded
4. 判定结果

LLM 版最终可输出三档结果：

通过：未发现明显结构问题，可进入下一轮判断
风险：存在 1–2 项中度问题，建议人工复核
拦截：存在明显 badcase 特征，不建议进入核心样本池
5. 推荐输出格式
{
  "language_naturalness": "pass",
  "over_explained": "mild",
  "setting_dependency": "low",
  "logic_coherence": "weakly_coherent",
  "information_density": "thin",
  "ai_trace": "suspicious",
  "backstory_overload": "light",
  "final_result": "risk"
}
6. 使用原则
LLM 版只用于识别“哪里有问题”，不直接判断“这是不是好句子”。
若文本在逻辑、AI 感、语言通顺度中出现明显 fail，应优先拦截。
若文本只是边界样本，不确定是否有味道，应交由人工判断版复核。
LLM 版适合做负向筛查，不适合做最终选优。
二、《ItClicks 高情绪短文本人工精选标准 v0.1（人工判断版）》
1. 目标

用于对通过初筛的文本进行人工复核，判断其是否真正成立、是否具有风格辨识度、是否值得进入核心样本池。
本版本的重点不是拦截错误，而是判断文本的保留价值。

2. 适用范围

适用于经过初步过滤后，需要进一步判断是否可沉淀为：

核心样本
正向风格样本
种子文本
prompt / style rag / 数据迭代参考样本

本版本聚焦短文本的成立度、辨识度和复用潜力，不评估剧情完整性或世界观丰富度。

3. 核心判断维度
3.1 情绪张力是否成立

判断文本中的拉扯、靠近、失衡、留白、悖论等是否自然成立。
不是看情绪词强不强，而是看关系动作和情绪结构是否真的让人感到“有那一下”。

人工重点看：

张力是否来自关系本身
是否有未说尽的部分
是否自然，而非硬拗
3.2 风格鲜明度

判断文本是否具有明确气质和表达个性，而不是退化成通用言情句式。
风格鲜明不等于夸张，而是能在冷感、克制、反差、位阶、悬停等方向上留下辨识度。

人工重点看：

是否一眼有气质
是否能与通用句式区分开
是否有持续沉淀价值
3.3 钩子感

判断文本是否在短时间内提供一个能被记住、继续改写、继续定制的抓手。
这个抓手可以是反差、异常措辞、悬停动作、时间切口、差一点的停顿。

人工重点看：

读完后是否能记住一句
是否让人想继续补全或延展
是否适合进入定制链路
3.4 高级自然度

判断文本虽然表面顺畅，但是否仍然带有“模型努力写得像文学”的姿态感。
这里关注的不是基础通顺度，而是更高阶的“像不像人会留下来的句子”。

人工重点看：

是否故作深沉
是否修辞太满
是否真的自然
3.5 可复用性 / 可迁移性

判断文本是否适合作为产品中的核心表达样本，而不仅仅是一句偶然成立的话。
优质样本应具备一定复用性，既不过度依赖特殊设定，也不会因为太空而失去个性。

人工重点看：

是否适合做种子文本
是否可以迁移到多个关系场景
是否具备产品价值而非单句偶得
3.6 总体保留价值

综合判断文本是否值得进入核心样本池。
这是人工版中的最终判定项，强调“这句有没有必要留下来”。

人工重点看：

是否能代表目标风格方向
是否能作为后续迭代参考
是否具备正样本价值
4. 判定结果

人工版建议使用三档结果：

淘汰：不成立或价值不足，不建议保留
可保留：基本成立，可作为普通参考样本
优先保留：成立度高、风格明确、钩子强，适合进入核心样本池
5. 人工判断参考问题

在复核时，可辅助使用以下问题：

这句的张力是自然成立的吗？
这句有没有明显的气质和辨识度？
这句读完后，脑中会留下一个抓手吗？
它是真的自然，还是只是表面流畅？
它适不适合被复用、沉淀、继续生成？
三、协同使用方式
1. 推荐流程

第一步：使用 LLM 判断版 进行批量初筛
第二步：拦截明显 badcase
第三步：将边界样本和通过样本交给 人工判断版 复核
第四步：人工选出可保留与优先保留样本
第五步：沉淀进入核心样本池，用于 prompt、style rag 和数据迭代

2. 分工原则
LLM 负责判断：这句有没有明显毛病
人工负责判断：这句到底有没有味道

也就是说：

机器做负向筛查
人工做正向选优
