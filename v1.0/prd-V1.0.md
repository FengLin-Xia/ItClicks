# PRD｜这也能代（It Clicks）1.0
> Web Demo（写作 IDE 的初始 demo）。两阶段体验：清透起始页（换一换/代一代）→ 画布阶段（软介入写作）。

---

## 1. 版本定位

- **形态**：Web Demo
- **核心主张**：用一段“可投射的引线文本”触发代入；用户愿意继续时进入画布，AI 以“建议/补全”的方式软介入。
- **不做**：账号/用户数据/作品保存/历史记录/分享/复杂 agent/风格库。

---

## 2. 目标用户与场景

### 用户
- 同人爱好者 / 代餐读者（不一定写，但会“代”）
- 轻度写作用户（想续两句、改一句，不想写长 prompt）

### 场景
- 打开网页：刷几段引线“换一换”
- 看到一段能代：点“代一代”进入画布，继续推一点点

---

## 3. 成功标准（1.0）

### 必达（定性）
1. seed 文本能让用户停顿并代入（哪怕很短）
2. 用户能理解“换一换/代一代”，并看得懂进入画布后的操作
3. AI 默认不覆盖用户正文，建议可插入/可放弃

### 可选埋点（不强制）
- 换一换次数、代一代转化率
- 画布页：继续/改写次数、建议采纳率（Accept rate）

---

## 4. 产品结构（两阶段）

### Stage 0｜起始页（清透）
**目标**：3 秒内完成一次“代入尝试”。

#### 页面元素
- 中央：seed 文本
- 底部按钮：
  - **换一换**：刷新 seed（覆盖显示）
  - **代一代**：进入画布页，并带入当前 seed
- 可选小字：`不用设定，读就行。`

#### 行为
- 页面加载自动生成一次 seed
- 换一换：重新请求 seed（可做 300–600ms 防连点节奏）
- 代一代：跳转画布页，seed 写入画布第一段

---

### Stage 1｜画布页（软介入写作）
**目标**：用户可以自己写；用户想要时，AI 给短段建议或改写备选。

#### 页面元素
- 全屏画布（textarea 推荐）
- 底部 Command Bar（非左右对半分）：
  - 输入框（可空）：一句名字/感觉/否定
  - 主按钮：**继续一段**
  - 次按钮（可选加分）：**改写选中**
- AI 建议卡片（输出区）：
  - **插入到光标/文末**
  - **替换选中**（改写）
  - **放弃**

#### 行为（1.0最小集）
- 进入画布：seed 自动写入，光标定位末尾
- 继续一段：
  - 取上下文（见 §7.3）
  - 生成 50–120 字短段
  - 以建议卡片展示，用户点插入后才写入画布
- 改写选中（可选）：
  - 无选中则提示“先选一句/一段”
  - 生成 2–3 备选改写，用户点选替换

---

## 5. 核心生成策略（1.0）

### 5.1 统一写作约束（北极星）
- **高信息密度 + 模糊指代 + 未完成感**
- 不追求“文学腔”，追求“同人代餐可代入的起点”
- 不强制“冲突”，只保证“关系状态不稳定/未确认”

### 5.2 Seed 原则
- 必须是**无前史独立片段**
- 禁止暗示具体已发生事件（避免“那天之后/自从那件事/后来才知道”）
- 不闭合、不解释、不定设定

### 5.3 Assist 原则（画布）
- 连续性仅保持在**语气与状态**，不补“剧情前情”
- 上下文只取局部（避免模型脑补整篇）

---

## 6. Prompt 方案（Writer + Critic 双调用）

> 目的：减少“端着/前史/解释/闭合”，提升稳定性。代价：token 近似翻倍（可接受）。

### 6.1 Writer Prompt（生成器）

#### System（固定）
你是一个“代入引线生成器”。

任务：生成一段可被同人爱好者代入的短文本，用作脑补或写作起点。

必须遵守以下规则：

【结构】
1. 输出中文，50–120 字，2–4 句。
2. 必须使用至少 3 个模糊指代（如：他、她、你、那句话、那件事、现在、最近）。
3. 不得使用具体背景设定（地名、职业、组织、世界观、具体事件名称）。
4. 不得解释原因，不得使用“因为/所以/其实/原来/终于/后来”等总结性语言。
5. 不得形成完整故事或明确结局。
6. 结尾必须停在“还可以继续”的位置（差一点/没说完/没确认）。

【语气】
7. 语气自然，像同人群里随手发的一段话；不要文学腔、不要宏大抽象词（命运/洪流/空气沉默等）。
8. 用动作/边界/选择表达张力，少用情绪形容词堆砌。

【禁止前史】
9. 不得暗示明确前史事件（禁止：那天之后/自从那件事/后来才知道/从此/最终/决定了）。

只输出文本，不要解释规则。

#### User（根据 mode 拼装）
- mode=seed：
  - “生成一段全新的独立引线文本。不要引用任何已有内容。”
- mode=assist, op=continue：
  - “基于下面的局部上下文，续写一小段（50–120字），保持语气与状态，但不要补前史、不要闭合。”
- mode=assist, op=rewrite：
  - “改写下面选中片段，给出 3 个候选版本（每个 30–80字），更自然、更好代入，仍遵守规则。”

> 注：rewrite 输出建议格式：用 `1) ... 2) ... 3) ...`，便于前端展示与选择。

---

### 6.2 Critic Prompt（守门员）

#### System（固定）
你是一个“代入起点评估器（critic）”。

目标：判断文本是否符合“高信息密度 + 模糊指代 + 未完成感”的代入引线标准，并拦截常见失败：
- 端着/文学腔
- 解释/总结/因果
- 前史暗示
- 结局感/闭合
- 设定绑定（地名职业世界观）
- 信息太空（只有形容词，没有关系动作）

请按标准输出 JSON，不要输出任何额外内容。

#### User（固定模板）
请评估下面文本：

【文本】
<<<TEXT>>>

输出 JSON：
{
  "pass": true/false,
  "problems": ["..."],
  "confidence": 0-1,
  "fix_hint": "一句话指出最重要的修改方向"
}

判定规则：
- 出现具体设定/前史暗示/解释总结/结局感 任一项 → 倾向 fail
- 若“端着”明显（宏大抽象词、刻意比喻、散文化）→ fail
- 若缺少未完成状态或缺少模糊指代 → fail

---

### 6.3 调用策略（双调用 + 轻重试）
- 先 Writer 生成
- 再 Critic 评估
- 若 fail：按 critic 的 fix_hint **重采样 1 次**（最多 2 次）
- 仍 fail：返回最后一次结果，但在 meta 标记 `quality=low`（便于你后续观察）

## 7. 技术方案（前端 HTML + 腾讯云同机后端代理）

### 7.1 为什么不能把 API 写在前端
- key 会被任何人从浏览器拿走（Network/源码）
- 无法限流，易被刷爆额度
- 不利于后续迭代 prompt/模型

结论：**必须做后端代理**，前端只请求你自己的 `/api/generate`。

---

### 7.2 部署形态（推荐）
腾讯云服务器上：
- Nginx：托管静态 HTML/CSS/JS
- Node 后端（Express/Fastify 任一）：提供 `/api/generate`
- DeepSeek key：放在服务器环境变量（或 `.env`，但不要进仓库）

同域好处：
- 同源请求简单（不折腾 CORS）
- `/api/*` 可由 Nginx 反代到 Node 端口

---

### 7.3 上下文截取策略（避免 token 膨胀和奇怪连续性）
- seed：不传历史
- assist：
  1) 若有选中：`selected`
  2) 否则：`around_cursor`（建议上下各 300–600 中文字符）
  3) 兜底：`tail`（最后 120–200 字）

**不要传全文**。

---

### 7.4 接口定义

#### POST `/api/generate`
Request：
```json
{
  "mode": "seed" | "assist",
  "op": "continue" | "rewrite",
  "hint": "（可空）一句名字/感觉/否定",
  "context": {
    "selected": "（可空）选中文本",
    "around_cursor": "（可空）光标附近截取",
    "tail": "（可空）末尾片段"
  },
  "state": {
    "tone": "restrained" | "intense"
  }
}

Response：

{
  "text": "生成文本（或 rewrite 的 1) 2) 3)）",
  "meta": {
    "retry": 0,
    "critic": { "pass": true, "confidence": 0.86, "problems": [] }
  }
}


---

7.5 最低限风控（必须）

Rate limit（按 IP）：例如 30 req/min（更保守也行）

输入上限：hint <= 200 chars；context 总长 <= 2000 chars

超限返回 429

日志：只记 mode/op/长度/耗时，不落用户文本（或仅在 dev 环境落）

8. 交付清单（1.0 Tonight）

Must Have

起始页：自动 seed、换一换、代一代

画布页：可编辑、继续一段（建议卡片：插入/放弃）

后端：/api/generate 代理 + DeepSeek key 在 env + 最低限流

Prompt：Writer + Critic 双调用 + 最多 1 次重试


Nice to Have

改写选中（3 个候选）

复制全文按钮

tone 切换（克制/更浓）



---

9. 产品判断口径（1.0）

Seed：不需要“写得漂亮”，只需要“代得动”

Assist：不需要“写完”，只需要“推一点”

任何端着/解释/前史/结局感，都视为失败信号，应通过 critic 拦截


---

### 本架构下的关键提醒（很实用）
- **别让前端直接打 DeepSeek**：就算你混淆 JS，也挡不住。
- 同机部署最省事：`/api/generate` 同域调用，不用 CORS；Nginx 反代即可。
