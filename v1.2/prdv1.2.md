
《这也能代》PRD V1.2

面向代餐圈层的 AI 情绪结构生成产品（概念定稿）


---

1. 产品定位

1.1 一句话定位

一个以情绪唤起为入口、以精准表达为出口的 AI 代餐生成产品。

1.2 是什么 / 不是什么

不是社区。
不是写作工具。
不是陪伴产品。

而是：

轻内容 Feed + 定制表达工具。

1.3 核心闭环

通过「刷流唤起情绪 → 自定义精准代入」完成表达闭环。


---

2. 目标用户

2.1 核心用户

代餐圈层用户

同人/代餐读者

有情绪投射需求、但不想写长 prompt 的用户

想要「差一点就说到心坎上」的精准表达

2.2 用户痛点（延续 V1.1 洞察）

内容平淡、张力不够

情绪解释过多、不克制

缺少「记忆点」、不够代

写长 prompt 太累

本版本解法：用刷流降低门槛，用「差一点」引导自定义。


---

3. 产品目标（V1.2）

3.1 用户侧目标

刷流：即时唤起情绪，制造「差一点」的轻度不满足

自定义：用 1–3 句输入，获得 80–120 字高张力代餐文本

可分享：导出卡片图片，完成表达闭环

3.2 产品侧目标

验证「刷流 → 自定义」转化路径

控制生成空间（语感、张力、结构压缩）

不依赖推荐算法（MVP 阶段）

成本可控、可讲商业逻辑


---

4. 整体结构（双层模型）

Layer 1：刷流（免费）— 入口与情绪唤起

Layer 2：自定义生成（API 调用）— 表达出口

```
┌─────────────────────────────────────────┐
│  Layer 1：刷流（免费）                    │
│  预生成内容池 · 无限滑动 · 卡片展示       │
│  情绪命中，但不完全贴合 →「差一点」       │
└─────────────────────────────────────────┘
                    ↓
            卡片底部引导：「试试生成你的版本」
                    ↓
┌─────────────────────────────────────────┐
│  Layer 2：自定义生成                      │
│  用户输入 → 结构抽取 → Prompt + RAG      │
│  → 80–120 字高张力文本 · 可导出卡片      │
└─────────────────────────────────────────┘
```


---

5. Layer 1：刷流层（入口）

5.1 功能清单

预生成内容池

无限滑动

卡片式展示

可分享图片

5.2 体验要求

即时加载（不能卡）

无推荐算法（MVP 阶段）

内容随机 + 结构均匀分布

5.3 核心策略

情绪命中，但不完全贴合。

作用：降低门槛、情绪唤起、制造「差一点」的感觉，引导进入自定义层。


---

6. Layer 2：自定义层（表达）

6.1 触发方式

卡片底部引导：「试试生成你的版本」

6.2 用户输入

自由描述（1–3 句）

或简单关系设定

6.3 系统流程

```
用户输入
   ↓
结构抽取
   ↓
Prompt 模板约束
   ↓
RAG 注入圈层语感
   ↓
高张力压缩输出
```

6.4 输出规格

80–120 字

不直白表达

高张力代餐文本

可导出卡片图片


---

7. Prompt 工程的作用

从「生成内容」升级为：控制生成空间。

作用包括：

保持语感统一

控制张力强度

避免风格漂移

结构压缩

稳定输出质量

它不消失，只是从前台玩法变成后台引擎。


---

8. 成本与变现逻辑

8.1 免费部分

刷流

基础生成次数

8.2 付费可能点（后续版本）

无限生成

高级结构模板

风格包

保存历史设定

8.3 本质卖点

精准表达权。


---

9. 产品核心指标（MVP 阶段）

优先级排序：

1. 刷流停留时间

2. 自定义点击率

3. 自定义完成率

4. 分享率

5. 二次生成率

不做：DAU 优先、推荐算法优先。


---

10. 核心哲学

不是做平台。
不是做创作者工具。

而是：

用刷流降低认知门槛，用轻度不满足引导表达。


---

11. 当前版本的优点

体验流畅（无 20 秒等待）

成本可控

技术结构清晰

可讲商业逻辑

可讲 AI 生成控制能力

不依赖推荐算法


---

12. 当前版本的风险

内容池规模过小会重复

刷流太爽会削弱定制

刷流太弱会流失

无推荐长期留存有限

注：以上属于 V2 讨论范围，MVP 先验证路径与指标。


---

13. 不做内容（范围控制）

社区与 UGC 平台

长篇写作工具

陪伴/聊天产品

推荐算法（MVP）

账号体系（可按需后置）


---

14. 版本总结

V1.2 是：

面向代餐圈层的 AI 情绪结构生成产品概念定稿。

核心卖点：

> 以情绪唤起为入口、以精准表达为出口；
> 刷流降低门槛，自定义完成表达闭环。

与 V1.1 的关系：从「关系驱动写作 IDE」收敛为「轻 Feed + 定制表达」，更聚焦代餐圈层与可落地 MVP。
