 这也能代 v1.1.3 — Index Seed Prompt Spec

🎯 目标

生成 2–3 句中文短文本。

文本必须：

围绕两人关系

发生于一次具体片段

不解释

不写人物设定

不写“会/总是/一向”等概括表达

不输出分析或说明



---

🧠 生成流程（后端逻辑）

Step 1 — 选择张力模式（核心驱动）

mode ∈ {
  core_action,   # 关系结构改变
  contrast,      # 轻微预期违背
  suspended      # 悬住型
}

建议权重（默认）：

core_action: 40%

contrast: 30%

suspended: 30%



---

Step 2 — 选择表达形式（外壳）

form ∈ {
  high_concept,   # 高概念句
  daily_scene,    # 生活片段
  emotional_line  # 情绪宣言
}

等概率即可。


---

🧩 不同张力模式的规则


---

① core_action 模式（关系爆点）

定义：

> 行为必须直接改变两人关系状态。



规则：

行为必须直接作用于对方

不得是外部社会行为（如谈判、合同、公司等）

必须是一次具体发生的片段


示例气质（不是模板）：

在她面前宣布某个不可回退的决定

把象征关系的物件处理掉

在对方最不该出现的时候出现


禁止：

人物背景说明

抽象设定



---

② contrast 模式（预期差）

定义：

> 第二句必须轻微违背第一句的自然推论。



规则：

不允许夸张反转

违背必须发生在两人关系内部

不可进入外部世界


例如：

说不见 → 却等

赶走 → 门没锁

删掉 → 密码没改



---

③ suspended 模式（悬住型）

定义：

> 第二句不解释、不反转，只增加新的物理或情绪维度。



规则：

可以增加环境

可以增加物理状态

可以增加未完成动作

不做总结


例如：

灯没关

笔帽没合

手机还在充电



---

🏷 表达形式约束


---

high_concept

可以使用概念角色（如正义/反派）

但必须落在具体行为上

不可写设定说明



---

daily_scene

优先使用日常空间

但不强制生活化



---

emotional_line

可含一句对话

不可使用长段抒情



---

🚫 全局禁止规则

必须加入：

- 禁止人物概括性描述（会、总是、一向、向来等）
- 禁止外部职业或社会宏大设定
- 禁止解释行为动机
- 禁止总结关系状态
- 不超过三句


---

📦 最终发送给 LLM 的系统 Prompt 模板

你后端拼接时类似这样：

你正在生成一段两人关系的短文本片段。

张力模式：{mode}
表达形式：{form}

请生成 2–3 句中文文本。

要求：
- 必须发生在一次具体片段
- 不写人物设定
- 不做解释
- 不超过三句
- 行为必须作用于两人关系内部

不同 mode 追加对应规则块。
